# TABIT

[한국어 README](./README.ko.md)

## プロジェクト紹介

TABITは、ユーザーの予算、旅行期間、旅行地域、人数、同行者タイプ、旅行スタイル、興味に合わせて、日本の旅行先と簡単な旅行日程を提案するAI旅行推薦Webサービスです。

最初はHTML/CSS/JavaScriptでプロトタイプを作成し、その後、Next.jsとTypeScriptを使用したWebアプリケーションへ移行しました。現在はNext.js API RouteからOpenAI APIを呼び出し、ユーザー入力に基づいた旅行推薦を生成します。

## 対象ユーザー

日本旅行を計画している旅行者を対象としています。

特に、旅行先を自分で調べる時間を減らし、予算、日程、旅行地域、興味に合った候補を素早く確認したいユーザーを対象としています。

## 主な機能

- 予算、旅行期間、旅行地域、人数、同行者タイプ、旅行スタイル、興味の入力
- 日本地域マップUIによる旅行希望地域の選択
- 出発日と帰国日を基準にした旅行期間の自動計算
- 予算、日付、人数、旅行地域、興味の入力バリデーション
- 韓国語 / 日本語 UI 切り替え
- OpenAI APIによる旅行推薦生成
- 選択した旅行条件の確認機能
- APIリクエスト中のローディング表示
- API失敗時のfallback推薦
- おすすめ旅行先、おすすめ理由、簡単な日程の表示

このプロジェクトでは、単にAI APIを呼び出すだけではなく、ユーザー入力、フロントエンドの状態管理、サーバーAPI処理、fallback処理、多言語UI、レスポンシブ対応の地域選択UIを一つのサービスとして実装することを目標にしました。

旅行業界、観光DX、Web開発、システムエンジニア職への応募を想定したポートフォリオプロジェクトです。

## 現在の処理フロー

    ユーザー入力
    -> React state管理
    -> 入力値バリデーション
    -> 選択した旅行条件の確認
    -> Next.js API Route
    -> OpenAI APIリクエスト
    -> AI旅行推薦生成
    -> 結果カード表示

OpenAI APIキーが設定されていない場合、またはAPIリクエストに失敗した場合は、選択した興味を基準にfallback推薦を返します。

## 環境変数

`frontend/.env.local` ファイルを作成し、OpenAI APIキーを設定します。

    OPENAI_API_KEY=your_api_key_here

`.env.local` はGitHubにコミットしないでください。

## 実行方法

    cd frontend
    npm install
    npm run dev

ブラウザで以下のURLにアクセスします。

    http://localhost:3000

## テスト項目

- 予算が入力されていない場合、エラーメッセージが表示されるか
- 最低予算未満の場合、エラーメッセージが表示されるか
- 出発日と帰国日が正しく設定されるか
- 帰国日が出発日より前の場合、エラーメッセージが表示されるか
- 旅行期間が自動で計算されるか
- 人数が1人の場合、同行者タイプの選択が非表示になるか
- 旅行地域が選択されていない場合、エラーメッセージが表示されるか
- 興味が選択されていない場合、エラーメッセージが表示されるか
- 推薦生成前に選択した旅行条件の確認ダイアログが表示されるか
- 推薦生成中にローディング表示が出るか
- OpenAI APIが成功した場合、AI推薦結果が表示されるか
- OpenAI APIが失敗した場合、fallback推薦が表示されるか
- 韓国語 / 日本語 UI の切り替えが可能か
- PC / モバイル画面で地域選択UIが正常に表示されるか
